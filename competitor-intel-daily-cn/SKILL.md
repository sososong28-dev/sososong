# Competitor Intel Daily CN

为“避孕套/安全套行业竞品情报日报”设计的可复用 OpenClaw skill。

当用户希望把一次性的竞品搜索，升级为**可重复执行的日报流程**时，使用本 skill。它覆盖采集、筛选、结构化整理、日报生成、消息摘要与定时执行，并默认面向中文业务团队输出。

## 适用场景

- 每日追踪避孕套/安全套行业与核心品牌动态。
- 按固定版式输出管理层可读的 Markdown/PDF 日报。
- 在 Windows 环境下，用 OpenClaw + 计划任务实现定时执行。
- 为产品、市场、渠道、管理层提供“风险提示 + 机会点”摘要。
- 扩展为周报、预警推送、专项监控任务。

## 快速参考

| 需求 | 动作 |
|---|---|
| 搭建日报 skill | 复制本目录并按 `config.json` / `keywords.json` / `sources.json` 调整 |

| 需要更强反爬能力 | 联合使用 `playwright-scraper-skill` 或浏览器自动化流程 |

## 工作流

### 1. 构建关键词矩阵

不要只搜品牌名，按“行业词 + 品牌词 + 组合词”建检索矩阵。

推荐顺序：

1. 行业主词：`避孕套`、`安全套`
2. 品牌词：`杜蕾斯`、`冈本`、`杰士邦`、`大象`、`名流`、`羽感`、`BeU`
3. 组合词：
   - `杜蕾斯 新品`
   - `冈本 超薄`
   - `杰士邦 聚氨酯`
   - `名流 直播`
   - `大象 小红书`
   - `羽感 天猫`
   - `BeU 京东`
   - `安全套 备案`
   - `避孕套 处罚`
   - `安全套 抽检`
   - `避孕套 行业报告`
   - `安全套 销量`
   - `避孕套 价格`
   - `安全套 投诉`
   - `安全套 召回`
   - `安全套 国家标准`

优先采集最近 24 小时；若关键栏目为空，可以回溯近 7 天。

### 2. 覆盖信息源

按下列优先级分层采集：

1. **国家/官方站点**：药监、抽检、标准、处罚、企业信息、知识产权、招投标。
2. **电商平台**：天猫、京东、淘宝、拼多多、抖音商城、快手商城、美团闪购、1688。
3. **电商第三方数据**：蝉妈妈、飞瓜、魔镜、灰豚、考古加、生意参谋公开片段。
4. **资讯站/新闻稿**：36氪、虎嗅、亿邦动力、电商报、行业垂类媒体。
5. **社媒线索**：小红书、抖音、知乎、公众号线索页。
6. **报告资料**：券商研报、行业白皮书、咨询报告、平台趋势报告、年报/招股书。

如果源站抓取失败：

- 记录失败原因；
- 在附录中标记“未成功抓取”；
- 不要编造缺失内容。

### 3. 标准化采集结果

每条信息统一整理为以下结构：

```json
{
  "date": "2026-03-19",
  "source_type": "电商平台/国家网站/资讯网站/调研报告/社媒",
  "platform": "天猫",
  "brand": "杜蕾斯",
  "keyword_hit": ["杜蕾斯", "新品", "天猫"],
  "title": "杜蕾斯某新品上线",
  "summary": "概括这条信息的核心内容，不超过120字",
  "url": "原始链接",
  "publish_time": "2026-03-19 10:22",
  "capture_time": "2026-03-19 18:30",
  "signal_type": "新品/价格/销量/评论/舆情/备案/处罚/报告",
  "importance": "high/medium/low",
  "credibility": "official/high/medium/low",
  "needs_review": false
}
```

最低保留字段：来源、发布时间、抓取时间、原始链接、分类、重要程度、可信度。

### 4. 去重与筛选

#### 去重规则

- 同一链接仅保留一次。
- 标题近似度大于 85% 视为重复事件。
- 多站转载时，仅保留：
  1. 官方源；
  2. 首发源；
  3. 信息最完整源。

#### 优先保留内容

- 新品上市或 SKU 扩充
- 价格带变化与促销强度变化
- 销量/排名变化
- 评论区高频好评与差评
- 处罚、抽检、备案、召回
- 大促、联名、礼盒、达人合作
- 行业报告中的关键数据点
- 值得产品、市场、渠道团队跟进的机会点

### 5. 输出日报

输出必须同时包含：

1. 今日核心结论
2. 品牌动态
3. 电商数据摘要
4. 官方/政策动态
5. 舆情与消费者声音
6. 风险提示
7. 机会点建议
8. 附录：来源链接、抓取时间、关键词清单、失败说明

不要简单堆链接；每个分节先给结论，再给证据。

### 6. 生成 PDF 与摘要

- 先输出结构化 Markdown。
- 再用统一模板转为 HTML/PDF。
- 文件命名：`避孕套行业竞品情报日报_YYYYMMDD.pdf`
- 同时产出 300-500 字消息版摘要，用于 Telegram、邮件、企业微信或钉钉推送。

## 输出规范

### A. 管理层版日报

必须简洁、商务、可扫描阅读。推荐页次结构：

1. **封面**：标题、日期、覆盖品牌、采集时间区间、一句话总结。
2. **今日核心结论**：3-5 条最重要发现。
3. **品牌动态**：按品牌列出动态、价格变化、新品/活动、评论反馈。
4. **电商数据摘要**：用表格展示平台、品牌、商品、价格带、促销、评论趋势、备注。
5. **政策/官方动态**：药监、抽检、备案、处罚、标准、知识产权、招投标。
6. **舆情与消费者声音**：好评点、差评点、争议点。
7. **机会点与风险提示**：机会、风险、建议动作。
8. **附录**：来源链接、抓取时间、关键词、去重说明、失败说明。

### B. 消息版摘要

建议结构：

1. 今日重点（3-4 条）
2. 电商观察（价格带、卖点、差评词）
3. 建议动作（2-3 条）

## 可信度与重要程度规则

### 可信度

- `official`：国家机关、品牌官方、平台官方、上市公司正式披露
- `high`：主流媒体、头部行业媒体、可靠第三方数据平台
- `medium`：一般资讯站、平台公开页面、样本有限的评论聚合
- `low`：转载站、线索页、二手整理、无法验证截图

### 重要程度

- `high`：处罚、抽检、召回、重大降价、爆款新品、重大舆情、平台排名显著变动
- `medium`：常规促销、内容种草、渠道动作、评论趋势变化
- `low`：普通转载、弱相关提及、无明确证据的讨论

## 难点处理

### 电商平台反爬

- 优先抓公开搜索页和详情页静态信息。
- 评论只抓样本，不做高频全量暴力采集。
- 尽量结合第三方数据平台。
- 增加缓存、限频、重试、代理。
- 需要浏览器自动化时，优先联动 `playwright-scraper-skill`。

### 官方站点稳定性

- 单独维护固定检索词模板。
- 给官方源更高优先级。
- 对失败原因写入附录，方便第二天排查。

### 真假识别与归因

- 默认官方优先。
- 无法判定首发时，选择信息最完整、时间最早的来源。
- 对“疑似同事件”保留一个主源，其他来源记为旁证。

## 周报与预警扩展

除日报外，建议扩展两类任务：

### 周报

每周汇总：

- 品牌动作变化
- 价格带趋势
- 高频差评词
- 有效卖点与机会点

### 预警

以下事件单独推送，不等日报：

- 国家抽检/处罚/召回
- 竞品大幅降价
- 爆款评论异常增长
- 某新品上线
- 某平台传播爆点

## 使用步骤

1. 根据业务范围修改 `keywords.json`。
2. 根据可访问源修改 `sources.json`。
3. 在 `config.json` 中设定回溯窗口、样本上限、输出目录和发送渠道。
4. 先用 `python scripts/generate_report.py --input examples/sample_items.json --date 2026-03-19 --skip-pdf` 做本地样例试跑。
5. 再按真实采集结果输入 `scripts/generate_report.py`，生成日报 Markdown、摘要和清洗结果。
6. 按 `templates/report_template.md` / `templates/summary_template.txt` 做版式或文案微调。
7. 在 Windows 中按 `references/windows-task-scheduler.md` 注册定时任务。
8. 如需抓动态页面或复杂站点，联用浏览器自动化 skill。

## 交付要求

输出结果必须满足以下原则：

- 不编造抓取失败的数据。
- 明确保留来源与时间。
- 结论优先，证据随后。
- 对风险、机会、建议给出明确指向。
- 允许栏目为空，但必须解释为空原因。
